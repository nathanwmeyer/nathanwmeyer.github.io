Senior Project

This project was for CST-451 and CST-452, part of the Bachelor of Science in Software Development at Grand Canyon University. The purpose of this project was to demonstrate planning and design capabilities.


Project Overview

Problem and Project Background

The purpose of this project is to develop an online game application that allows users to play a puzzle game where internet access is available. The goal is to provide entertainment for users by developing an application that is easy to access and use.

Application of Christian Worldview

According to the Christian worldview we as Christians follow the doctrine given to us by the word of God in the Bible. The Bible tells us to be kind to others, recommending that we show hospitality to strangers in passages such as Hebrews 13:2 (English Standard Version Bible, 2001). Showing kindness and love to others is one of the main ethical perspectives of the Christian Worldview, and this aspect is what I plan to express in this project.

Project Objectives

The objective of this project is to build a web application that can be accessed by users. The users will be able to log in to play the game and can save the games they are playing to their logged-in account.

Project Challenges and Measures of Success

The project will meet six deadlines during its development. The project's success is measured by what percentage of goals had been met before each deadline.

Benefits and Opportunities

This project will allow all participating developers to gain experience in the web application development field and demonstrate knowledge of common development practices.

Project Summary

This project is a website where users can register and login to play a game based on Lights Out. The user can choose to start a new game, save a game in progress, and resume a saved game at any time. Both user login information and saved game data are stored securely in a relational database.

Game Logic Class Change Button Method

// method: changeButtons. This method changes the button board based on the button clicked. This method returns the updated list of buttons. public List changeButtons(List buttons, int buttonId) { // retrieve the current value of the selected button int value = buttons.ElementAt(buttonId).State; // change the button's state buttons.ElementAt(buttonId).State = (value + 1) % 2; // get the information about the surrounding buttons // is the button on the left a valid button to change? bool clearLeft = buttonId % ROW_LENGTH > 0; // it will only be valid if this button is NOT the first button // is the button on the right a valid button to change? bool clearRight = buttonId % ROW_LENGTH != ROW_LENGTH -1; // it will only be valid if this button is NOT the last button // is the button above this button a valid button to change? bool clearTop = buttonId > ROW_LENGTH - 1; // it will only be valid if this button is NOT on the first row // is the button below this button a valid button to change? bool clearBottom = buttonId < GRID_SIZE - ROW_LENGTH; // it will only be valid if this button is NOT on the last row. //change the buttons that are valid if (clearLeft) { buttons.ElementAt(buttonId - 1).State = (buttons.ElementAt(buttonId - 1).State + 1) % 2; } if (clearRight) { buttons.ElementAt(buttonId + 1).State = (buttons.ElementAt(buttonId + 1).State + 1) % 2; } if (clearTop) { buttons.ElementAt(buttonId - ROW_LENGTH).State = (buttons.ElementAt(buttonId - ROW_LENGTH).State + 1) % 2; } if (clearBottom) { buttons.ElementAt(buttonId + ROW_LENGTH).State = (buttons.ElementAt(buttonId + ROW_LENGTH).State + 1) % 2; } // return the updated list of buttons return buttons; }

Project Background Details

The purpose of this project is to develop an online game application that allows users to play a puzzle game where internet access is available. The goal is to provide entertainment for users by developing an application that is easy to access and use.


Approach to Implementation

General Technical Approach:

This project is a web-based application that can be used by any user through a web browser. When the user first accesses the website, they will be given the choice to log in or register a user account. If they attempt to access the game page without logging in, they will be redirected to the registration page instead. When registering, the user must provide a username and password, which will be stored securely on a MySQL server. After logging in or registering, the user will be allowed to play the game. While they are playing the game, the user can save the game, load a previously saved game, or reset the game board with a new game.

Implementation of Non-Functional Requirements

For the storage and management of data, we used MySQL for the Database management software. In addition, BCrypt.Net-Next was implemented to allow for the secure hashing of passwords. Finally, C# .NET was utilized to allow for the usage of Razor webpages and C# classes in the creation of the application's classes.
